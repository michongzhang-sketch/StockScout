description: 对单只股票或股票池执行标准化筛选、评级和结果整理。
---

# 适用场景

- 用户给出一个股票池，希望挑出优先研究对象
- 用户只给出行业、主题或市场，希望建立观察名单
- 用户需要把筛选结论沉淀为可复盘的结构化结果

# 输入

- 标的范围：股票代码、股票池、行业、主题或市场
- 偏好约束：价值 / 成长 / 红利 / 趋势 / 波段等风格
- 风险要求：回撤容忍度、是否接受高估值、高波动或单一赛道集中
- 输出要求：表格、Markdown 摘要或需要落盘到 `tmp/`

# 执行步骤

1. 明确筛选目标和时间周期。
2. 定义硬性过滤条件，先排除明显不符合要求的标的。
3. 对剩余标的按以下维度逐项比较：
   - 行业地位与业务质量
   - 盈利能力与成长性
   - 估值、安全边际与预期透支程度
   - 趋势、支撑压力位与入场性价比
   - 财报、政策、周期、流动性等风险
4. 给出分组结果：
   - 候选
   - 观察
   - 回避
5. 如果需要产出文件，把结果写入 `tmp/`，并保留时间戳与筛选前提。

# 输出要求

输出结果至少包含：

- 筛选前提
- 过滤规则
- 候选标的及核心理由
- 回避原因
- 风险提示
- 后续跟踪指标

# 质量标准

- 每个结论都要对应理由，不能只有标签没有依据。
- 不确定的信息要显式标注为“待验证”。
- 结论应优先帮助用户缩小研究范围，而不是直接替代投资决策。
